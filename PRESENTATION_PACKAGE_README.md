## 📸 Screenshot Capture Guide

### **Why Screenshots Matter**
Even if you do a live demo, have backup screenshots in case of:
- Internet connectivity issues
- Application errors
- Time constraints
- Nervous presentation jitters

### **How to Capture (macOS)**
1. Start your DOT Corridor Communicator application
2. Set browser zoom to 100%
3. Hide browser toolbars: View → Hide Toolbar (or F11)
4. Use screenshot tools:
   - `Cmd+Shift+4` (select area) - Best for specific features
   - `Cmd+Shift+3` (full screen) - For full application view
5. Screenshots save to Desktop by default
6. Rename systematically: `screenshot_ipaws_01.png`

### **Priority Screenshots (in order)**
Follow the detailed guide in `DIAGRAM_SPECIFICATIONS.md` section "Screenshot Capture Guide"

**Top 7 Must-Have Screenshots:**
1. Main map with active interstate incidents
2. V2X deployments layer with info panel
3. IPAWS alert generator (full screen)
4. Population impact detail view
5. CADD/IFC model viewer
6. Work zone with corrected geometry badge
7. CADD elements on map with feedback panel

### **Inserting Screenshots into PowerPoint**
1. Open presentation to Slide 11
2. Delete placeholder text
3. Drag-and-drop PNG files into slide
4. Resize and arrange (2-3 screenshots per slide works well)
5. Add brief captions if needed

## 🎤 Presentation Best Practices

### **Timing (Total: 12-15 minutes)**
- Slides 1-8: 8-10 minutes (core content)
- Slides 9-10: 2-3 minutes (gap analysis & recommendations)
- Slide 11: 5-7 minutes (demo) OR skip if no demo
- Slide 12: Q&A (time permitting)

**Pro tip:** If running short on time, skip Slide 11 (demo) and mention "Happy to show the live platform during Q&A or breaks."

### **Delivery Tips**
✅ **DO:**
- Start strong: "Thank you for the opportunity to present our Interstate 2.0 capabilities..."
- Use federal data sources as credibility: "This integrates USDOT V2X data, FHWA ARNOLD..."
- Connect to Interstate 2.0 vision: "This demonstrates the technology exists today..."
- Show enthusiasm about the platform
- Make eye contact (don't read slides)
- Pause for emphasis on key points

❌ **DON'T:**
- Apologize for UI/UX issues unless blocking
- Get defensive about constraints (Slide 7)
- Go over time - respect the 10-15 minute limit
- Read bullet points word-for-word
- Use jargon without explanation

See `DEMO_SCRIPT.md` section "Q&A Preparation" for:
- Anticipated questions with prepared answers
- How to deflect questions outside your expertise

**Application:**
- [ ] Start backend: `node backend_proxy_server.js`
- [ ] Start frontend: `cd frontend && npm start`
- [ ] Verify loads at `http://localhost:3000`

**Computer Setup:**
- [ ] Close email/Slack/messaging apps
- [ ] Enable "Do Not Disturb" mode

- [ ] Test screen sharing in meeting software
- [ ] Have backup screenshots ready (just in case)
- [ ] Clear browser cache if app is slow
- [ ] Disable browser notifications

**Presentation Room:**
- [ ] Test projector connection
- [ ] Verify internet connectivity
- [ ] Have backup plan if WiFi fails (mobile hotspot?)
- [ ] Test audio if presenting remotely

### **Backup Plan**
If live demo fails:
1. Don't panic - smile and say "Let me show you screenshots instead"
2. Switch to backup screenshots on Slide 11
3. Narrate what you would have shown
4. Offer to do live demo during Q&A or break
5. Total time to switch to backup: <15 seconds
